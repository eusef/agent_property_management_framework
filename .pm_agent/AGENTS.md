# PM_FRAMEWORK — Agent Entry Point

## Start Here

1. **Read `.pm_agent/AGENT_CONTEXT.md`** — Primary context file with project overview, file map, workflows, and approval thresholds
2. **Read `.pm_agent/AGENT_SETUP.md`** — Detailed working instructions, CSV schemas, session hygiene
3. **Read `TASKS.md`** — Current active tasks and priorities
4. **Read `DASHBOARD.md`** — Portfolio overview and status snapshot

## Project Structure

├── compliance/                 # Jurisdiction-specific legal summaries

│   ├── app/services/           # config_svc, onboarding_svc, dashboard_svc

## Key Rules

- **All data is markdown + CSV** — no databases, no cloud dependencies
- **Review-then-send** — Agent drafts, owner approves all outbound communications
- **Approval thresholds** — See AGENT_CONTEXT.md for detailed thresholds
- **Property READMEs use Field|Value tables** — Parse them for structured data
- **config.json** — Read/write via `webapp/app/services/config_svc.py`, not directly

## Web Dashboard

The webapp runs on `http://localhost:8000`. Launch with `START.command` (macOS) or `cd webapp && python -m app`.

- First launch opens a browser-based setup wizard at `/onboarding`
- Settings available at `/settings` (gear icon in header)
- Installable as a PWA (desktop app) via Chrome

## Webapp Development

- **Framework:** FastAPI + HTMX + Jinja2 (no JS framework)
- **Tests:** `cd webapp && python -m pytest tests/`
- **Requirements:** Python 3.11+, dependencies in `webapp/requirements.txt`
- **Config:** `webapp/app/config.py` — repo root auto-detected from webapp location
- **Excluded from file browser:** webapp, .git, .agents, .venv, __pycache__, .DS_Store, .claude, .pm_agent
